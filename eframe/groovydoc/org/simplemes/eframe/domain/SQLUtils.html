        • <T> @java.lang.SuppressWarningsSuppressWarnings List executeQuery(java.lang.String sql, java.lang.Class domainClass, java.lang.Object[] args)

          Executes the given SQL as a prepared statement with the given arguments. The row offset/limit defaults to offset 0, 100 rows max for non-COUNT SELECT statements.

          Note: This method can detect the SQL 'IN(?)' scenario when an array is passed in. This will expand the 'IN(?)' to 'IN(?,?,?...)' for the given array size. This can make the SQL much longer than the original.

          Parameters:
          sql - The SQL.
          domainClass - The domain class to bind the result set to or Map. If Map, then the results will be a list of Maps. The Map element names are the lower-case form of the column names from the DB.
          args - Optional arguments for the query. The first element can be a Pageable for row limits.
          Returns:
          The list of records found.

        • Tuple2 fixInClauseArray(java.lang.String sql, java.lang.Object[] args)

          Fixes the IN(?) clause for an SQL statement. Expands the array in the arguments into individual elements in the args list. Also expands the IN(?) to IN(?,?,?)...

          Parameters:
          sql - The SQL to alter.
          args - The arguments.
          Returns:
          The tuple with the new SQL and new args list.
