Package: org.simplemes.eframe.search

[Groovy] Class SearchHelper

    • org.simplemes.eframe.search.SearchHelper

    • This helper encapsulates the search interface. This allows for easier use of the search engine and supports unit testing with an interface that can be easily mocked. This helper provides a background indexing mechanism to reduce the affect on the application's domain record updates.

      This helper class can be created once when the application starts up and should never need refreshing.

      Note: This is an internal helper class. Application and module code should use the org.simplemes.eframe.search.service.SearchService instead. This class is subject to change.

      Logging

      The logging for this class that can be enabled:
      • info - Performance timing.
      • debug - Debugging information. Typically includes inputs and outputs for each normal (non-bulk) request.
      • trace - Detailed information on bulk actions. Ths includes the record IDs for the objects indexed.

        • @groovy.transform.CompileDynamicCompileDynamic SearchResult domainSearchInDB(java.lang.Class domainClass, java.lang.String search, Map params)

          Performs a domain search via SQL with the query string. This uses a key-based SQL search.

          params

          The params parameter can contain these options:
          • max/offset/sort/order - The standard paging/sorting parameters from the controller.
          • options - See below.

          options

          The options parameter can contain these options:
          • postProcessor - A closure that is execute for each record found. Useful for adding/removing elements from the list for each row.
          • additionalProperties - A list of property names (strings) to include in the formatted output. This allows you to display transients in the list output.
          Parameters:
          domainClass - The domain class to search.
          search - The query string. Optional. If not provided, then finds all records.
          params - Optional query parameters. Supported elements: offset/max and sorting options as supported by the ControllerUtils. Also supports an options element for other ControllerUtils features (see Options above).
          Returns:
          The search result, containing the list of values found.
