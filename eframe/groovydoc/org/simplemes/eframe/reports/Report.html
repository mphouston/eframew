      • Method Detail

        • java.util.Map<java.lang.String, java.lang.Object> buildBaseParametersForReport()

          Builds the base input parameters to pass to the report. This mainly builds values such as BaseDir the the resource bundle needed by most reports.

        • java.lang.Integer determineParameterSequence(java.lang.Object parameter)

          Determines the effective display sequence to use for the given report engine parameter. Uses the custom property 'sequence' if available. Otherwise, uses the documented Parameter Display Sequence from the framework Guide.

          Parameters:
          parameter - The parameter to determine the display sequence for.
          Returns:
          The display sequence. Never null.

        • java.util.Map determineReportParams(java.util.Map inputParams)

          Determines the effective report parameters to use. Check the URL (controller) params and the user preferences as a fall-back. This uses the controller params and adds any parameters from the user preferences that are not in the controller params.

          Parameters:
          inputParams - The parameters from the URL.
          Returns:
          The effective parameters.

        • java.lang.String getBundleRelativePath()

          Returns the relative path for a bundle reference. Will adjust the path to a resource classloader relative path if from a .jar file.

          Returns:
          The adjusted bundle path to use to load the bundle.

        • java.lang.Object getCompiledReport()

          Returns the compiled report. Compiles the report, if needed.

          Returns:
          The compiled report.

        • java.util.Map<java.lang.String, java.lang.Object> getDefaultParameters()

          Finds the default values for the report parameters.

        • java.util.List getDefinedReportParameters()

          Finds the defined report parameters.

          Returns:
          A list of JRParameter objects.

        • java.lang.Integer getEffectiveRowLimit()

          Determine the effective row limit to use for the report. Can be overridden by the application.yml setting: org.simplemes.eframe.report.rowLimit

          Returns:
          The row limit.

        • java.util.List<ReportFieldDefinition> getReportParameters()

          Finds the effective report parameters to use. This uses the controller params and the default values from the compiled report. List is sorted in display sequence.

        • java.util.Map<java.lang.String, java.lang.String> getSubReports()

          Returns the list of sub-reports that need to be compiled. Finds parameters that start with 'SubReport' and extracts the report name from the default value expression. The default value must be in the form: '$P{BaseDir}+"/SampleSubReport.jrxml"'.

          Returns:
          A map of the sub-report. The key is the parameter name for the sub report.

        • boolean isPdf()
