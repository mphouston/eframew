    • org.simplemes.eframe.preference.service.UserPreferenceService 

    • @groovy.util.logging.Slf4j
@groovy.lang.Singleton
class UserPreferenceService
extends java.lang.Object

      Provides access to the user preferences that are used to save GUI settings and related objects. This is the main business logic for the preference handling. All logged in users can store their preferences.

        • @javax.transaction.Transactional java.util.Map findPreferences(java.lang.String pageURI, java.lang.String desiredElement, java.lang.String preferenceType)

          Finds the given preference(s). This is designed for use in the browser for dynamic preferences such as dialog sizes.

          Note: Reads via the second level cache.

          Parameters:
          pageURI - The URI for the page these settings are to be found for (Required).
          desiredElement - The ID of the element for the preference (Optional).
          preferenceType - The preference type to return. For example, 'DialogPreference' (Optional).
          Returns:
          The list of matching preference(s).

        • @javax.transaction.Transactional java.lang.Object findSimplePreference(java.lang.String pageParam, java.lang.String elementParam)

          Saves the given preference value into the user's preferences for the give element. This is a simplified view of preferences that just supports a simple name/value setting.

          Note: Reads via the second level cache.

          Parameters:
          pageParam - The page (URI) the preference applies to.
          elementParam - The page element (or preference name).
          Returns:
          value The value. Can be a POGO or a simple element.
